JavaSE笔记

编码规范

空行

  1. 类声明和接口声明之间保留两个空行。

  2. 两个方法之间保留一个空行。

  3. 方法的第一条语句之前保留一个空行。

  4. 代码注释（尾端注释外） 之前保留一个空行。

  5. 一个方法内的两个逻辑段之间。

空格

  1. 赋值符号“=”前后各有一个空格。

  2. 所有的二元运算符都应该使用空格与操作数分开。

  3. 一元操作符： 负号“-”、 自增“++”和自减“–”等， 它们与操作数之间没有空格。

  4. 小左括号“(”之后， 小右括号“)”之前不应有空格。

  5. 大左括号“{”之前有一个空格。

  6. 方法参数列表小左括号“(”之前没有空格， 小右括号“)”之后有一个空格， 参数列表中参数逗号“,”之后也有一个空格。

  7. 关键字之后紧跟着小左括号“(”， 关键字之后应该有一个空格。

缩进

  1. 在方法、 Lambda、 控制语句等包含大括号“{}”的代码块中， 代码块的内容相对于首行缩进一个级别（4个空格） 。

  2. 如果是if语句中条件表达式的断行， 那么新的一行应该相对于上一行缩进两个级别（8个空格） ， 再往后的断行要与第一次的断行对齐。

断行

  1. 在一个逗号后面断开。

  2. 在一个操作符前面断开， 要选择较高级别的运算符（而非较低级别的运算符） 断开。

  3. 新的一行应该相对于上一行缩进两个级别（8个空格） 。

其他

  1. 在声明变量或常量时推荐一行一个声明。

  2. 每行至多包含一条语句。

  3. if、 for等控制语句只有一行代码情况下 ，不建议省略大括号。

驼峰命名法

常量

  • 一个单词：全部大写 IP

  • 多个单词：全部大写，单词之间以下划线分割 MAX_VALUE

变量 方法

  • 一个单词：全部小写 number

  • 多个单词：第一个单词首字母小写，其余单词首字母大写localVariable

类 接口

  • 所有单词首字母大写 ThreadPoolExcutor

包

  • 全部小写，域名反转 com.cskaoyan.name

进制

二进制 前缀0b或0B，还可以加下划线使其更易读 0b1001_1001

八进制 前缀0（易混淆，不建议用八进制）

十进制 无

十六进制 前缀0x或0X

计算机编码

原码：加符号位，正0负1

反码：原码的非符号位按位取反（原码到补码转换的中间值）

补码：正数就是其原码，负数为其非符号位按位取反后+1

数据类型

有8种基本类型：4种整型，2种浮点型，1种字符型，1种布尔型

有3种引用数据类型：类，接口，数组（class, interface, []）

整型（默认int）

  • byte 1字节 -128 ~ 127

  • short 2字节 -32768 ~ 32767

  • int 4字节 刚刚超过20亿

  • long 8字节 long型常量后缀尽量写大写L，避免与1混淆

浮点（默认double）

如果采用十进制表示指数， 需要使用大写或小写的e表示

幂， e2表示10^2。如 3.14e2 表示3.14*10^2

  • float 4字节 6-7位有效位 常量后缀写大写F

  • double 8字节 约15位有效位

字符

char c1 = ‘A’;

char c2 = ‘\u0041’; Java字符采用双字节Unicode编码， 占两个字节（16位） ， 因而可用十六进制（无符号的） 编码形式表

示， 它们的表现形式是\un， 其中n为16位十六进制数， 所以’A’字符也可以用Unicode编码’\u0041’表示

数据类型转换

  • boolean类型的值不能直接转化为其它数据类型的值

  • 默认(自动)的转化 byte,short,char—>int—>long—>float—>double byte,short,char相互之间不转换，他们参与运算首先转换为 类型

  • 强制转化（截去高位） 目标类型 变量名=(目标类型)(被转换的数据);
